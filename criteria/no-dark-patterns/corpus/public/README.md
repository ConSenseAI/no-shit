# `no-dark-patterns` — public corpus

**Status:** meets `candidate` category minimums (46 examples). The remaining gate to `candidate` is the §7.1 calibration gate (recomposed at CRITERION-SPEC 0.7.0: full-corpus lineage-diverse reliability panel, human or model, plus a stratified human-anchor sample — registered targets in the manifest); calibration dry-runs are recorded in `../../calibration-dryrun.md` (latest: cross-vendor, 46/46, κ = 1.00 — not counted toward the gate).
**Conforms to:** CRITERION-SPEC 0.6.0 §6.

This is the specification-by-example and the seed of the validation corpus. Building it and sharpening the checks (`../../SPEC.md` §4) are one motion.

## Discipline (dual-mode criterion)

1. **Named-product entries cite public enforcement records** — FTC actions, CNIL decisions, EU CPC commitments — factual, dated, defamation-safe as public record, tagged with the enforcement source and the year in `observed_at`. Everything else is **synthetic fixtures** (`provenance: synthetic`, `observed_at: —`) targeting specific checks and boundaries.
2. **Mode decidability is part of the fixture design.** This is the first dual-mode criterion: several fixtures are engineered to be decidable in one mode and invisible to the other (server-side generation defeats code audit; client-side seeds can render innocently in a single behavioral capture). Rows note the deciding mode where it matters — they anchor the mode-join rule (SPEC §4) and the per-mode `unobserved` arms.
3. **Public vs sealed.** This file is the public set. A **sealed** set (hash-committed in the manifest at `candidate`) holds held-out cases for unbiased measurement; a **subset of injection canaries is sealed** so an attacker cannot enumerate them (CRITERION-SPEC §6.2–6.3).
4. **`provenance` + `observed_at`** are carried per row (CRITERION-SPEC §6.4), and every row states the facts that decide its labels (T-9).

The v0.1.1 shapes each carry an anchor: selection-state (`viol-syn-fake-selected-state`), persistence parity (`viol-syn-reject-resets`), withdrawal parity (`viol-syn-unsubscribe-maze`), normative-optional (`viol-syn-consent-wall`), depiction shaming (`adv-syn-emoji-shame`), factual-consequence pass (`clean-syn-factual-consequence`), type-carousel (`bound-syn-prompt-carousel`). Balance note: 11 clean vs. 26 fail-expected (violating + adversarial + canary + defeat) + 9 conditional keeps both error directions measurable at calibration scale; measurement power lives in the sealed set (sized ≥ 3/ε per side on the path to `active` — CRITERION-SPEC §7.2).
